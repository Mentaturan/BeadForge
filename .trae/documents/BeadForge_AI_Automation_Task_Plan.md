# BeadForge 自动化代码迭代任务计划

## 📋 任务概述

创建一个每小时运行的自动化任务，通过以下流程持续改进 BeadForge 项目：
1. 分析代码 → 2. 搜索同类项目学习 → 3. 识别不足 → 4. 改进代码 → 5. 测试验证 → 6. 如遇bug回滚重试 → 7. 提交所有变更

## 🔄 自动化任务执行流程

### 阶段1：准备阶段（安全检查）
1. **创建备份提交**
   ```bash
   git add -A
   git commit -m "Backup before AI iteration - $(date)"
   ```
2. **检查是否有未提交的变更**
3. **记录当前版本SHA**

### 阶段2：代码分析
1. **静态代码分析**
   - 识别代码结构（HTML结构、CSS样式、JS逻辑）
   - 找出潜在问题（性能瓶颈、代码异味、错误处理）
   - 分析架构模式
2. **生成分析报告**（保存到 `automation_log/`）

### 阶段3：同类项目研究
1. **搜索开源同类项目**
   - 使用 WebSearch 搜索相关技术
   - 分析其他拼豆工具、设计软件的开源实现
2. **提取可借鉴的模式**
   - UI/UX 设计模式
   - 性能优化技巧
   - 代码架构方案

### 阶段4：识别不足与改进
1. **对比分析**
   - 当前代码 vs 优秀实践
   - 识别可改进点（按优先级排序）
2. **制定改进计划**
   - 优先级1：性能和稳定性改进
   - 优先级2：代码质量改进
   - 优先级3：功能增强

### 阶段5：实施改进
1. **安全修改策略**
   - 每次只做一个小改动
   - 改动后立即测试
   - 失败则立即回滚
2. **测试驱动改进**
   - 自动化测试验证
   - 浏览器控制台错误检查
   - 功能完整性检查

### 阶段6：验证与回滚机制
1. **测试验证**
   - 运行基础功能测试
   - 检查HTML语法
   - 检查JavaScript语法
2. **回滚策略**
   - 如果测试失败：`git reset --hard <backup_sha>`
   - 记录失败原因
   - 跳过该改进点，继续下一个

### 阶段7：提交与记录
1. **生成执行报告**
   - 记录每个步骤的结果
   - 保存改进内容
   - 记录执行时间
2. **Git提交**
   ```bash
   git add -A
   git commit -m "AI iteration: [改进内容摘要]"
   ```
3. **上传执行日志**到仓库

## 📊 自动化脚本设计

### 核心脚本结构
```
/workspace/
├── scripts/
│   ├── analyze_code.sh           # 代码分析脚本
│   ├── run_tests.sh              # 测试执行脚本
│   ├── rollback.sh              # 回滚脚本
│   └── ai_iteration.sh          # 主迭代脚本（由调度器触发）
├── automation_log/
│   ├── analysis/                # 分析报告
│   ├── improvements/            # 改进记录
│   └── test_results/            # 测试结果
└── .ai_context/                  # AI上下文存储
    ├── last_backup_sha.txt       # 上次备份SHA
    ├── current_version.txt       # 当前版本信息
    └── iteration_history.json    # 迭代历史
```

### 主迭代脚本逻辑
```bash
#!/bin/bash
# ai_iteration.sh

# 1. 准备阶段 - 创建备份点
./scripts/prepare_backup.sh

# 2. 分析当前代码
./scripts/analyze_code.sh

# 3. 研究同类项目
./scripts/research_similar_projects.sh

# 4. 实施改进
./scripts/improve_code.sh

# 5. 测试验证
./scripts/run_tests.sh

# 6. 如果成功则提交，否则回滚
if [ $TEST_PASSED -eq 0 ]; then
    git_commit_changes.sh
else
    ./scripts/rollback.sh
fi

# 7. 生成报告
./scripts/generate_report.sh
```

## 🧪 测试策略

### 基础测试
1. **HTML语法检查** - 使用 `htmlhint` 或基础解析
2. **JavaScript语法检查** - Node.js 语法验证
3. **JSON测试文件验证** - 确保测试数据格式正确

### 功能测试
1. **文件完整性检查** - 验证HTML文件可被浏览器解析
2. **脚本可执行性** - 检查JS代码无语法错误
3. **资源完整性** - 验证所有引用的资源（如果有）

### 回归测试
1. **备份点对比** - 确保修改确实发生了
2. **文件大小监控** - 检测异常变化
3. **关键功能标记** - 标记必须保留的功能

## ⚠️ 安全机制

### 回滚触发条件
- 测试失败
- 语法错误
- 功能损坏
- 运行时错误
- 异常大的文件变化

### 回滚执行
```bash
# 从备份SHA恢复
git reset --hard $BACKUP_SHA
git clean -fd
```

### 限制机制
- 每次迭代最多3次尝试
- 每次修改不超过500行
- 冷却期机制（如果连续失败）

## 🎯 关键设计决策

1. **稳定性优先**: 任何bug立即回滚
2. **小步快跑**: 每次只做小幅改进

4. **安全第一**: 先备份再修改
